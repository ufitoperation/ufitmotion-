Field Dictionary & Data Entry Guide

Organizations top-level tenant boundary

The top of the hierarchy. Every other row in the system is scoped to one of these. Typically created once per district when Ufit signs a new client.

Add via: Admin → Settings → Organizations POST /api/organizations

FieldTypeRequired?Notes
organization_idintAutoPrimary key, auto-incremented.

organization_typetextRequiredOne of: school_district, charter_network, private_school, independent_school.
billing_contacttextOptionalName + email of the district CFO or AP for contracts. Free text.
contract_statustextRequiredOne of: active, pending, paused, terminated. Default active.
created_attimestampAutoSet on insert.

Why local_student_identifier matters. Parents self-register by entering their child's name + this identifier. We verify both match. Without the identifier, parents have to know an internal integer ID — most won't. Always populate this field if your school provides it on the roster.

Coach Evaluations formal periodic reviews

Add via: Admin → Evaluations → + Formal Evaluation

Long-form evaluation of a coach over a period (typically quarterly). Includes the rolling score, narrative feedback, action plan, and goals for the next period.

Principal Satisfaction Surveys

Add via: Organization Portal → Surveys → + New Survey

Quarterly principal feedback — overall satisfaction, perceived program impact, communication quality, suggestions. Read-only for admins; principals own the entry.

MethodWhen to useExample

Bulk Coach Invite (Admin → Coaches → Bulk Invite)

Format per row: first_name,last_name,email,role

Defaults: If role is omitted on a row, the modal's "Default Role" dropdown applies. If school_ids not provided per row, the "Default Schools" multi-select applies.

Allowed roles: head_coach, assistant_coach, site_coordinator, coach_overseer, principal, school_staff. Admin / CEO are NOT allowed via bulk invite (privilege-escalation guard).

Cap: 200 rows per batch. If you need to invite more than 200 coaches at once, split into multiple batches. The server returns per-row results so you can see exactly which row failed (if any).
